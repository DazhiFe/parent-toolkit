# 首页常用网站功能增强计划

## 摘要

将首页现有的 6 个硬编码"常用网站"卡片，改造为**数据驱动、分类展示、访问频率智能排序**的推荐网站模块，涵盖教育、工具、资讯、学习、亲子等 8 大分类、约 40+ 个精选网站，并通过 localStorage 记录点击频次实现轻量个性化排序。

## 现状分析

- **HomeView.vue**："常用网站"区硬编码 6 个卡片（教育部、人教网、国家中小学智慧教育平台、慕课平台、国家图书馆、科普中国），无分类、无排序
- **SideNav.vue**：首页侧边栏有 `websites` 导航项，指向该区域
- **StudyAssistantView.vue**：已有数据驱动 + `v-for` 循环卡片 + 颜色/图标映射的成熟模式，可参考复用
- **useToast.js**：已有的轻量通知组件

### 完整网站数据（约 35-40 个）

每个网站包含字段：`{ id, name, url, description, category, color, icon, tag? }`

其中 `tag` 为推荐标签，如 `"热门"` `"官方"` `"免费"` 等，用于新用户引导。

## 访问频率排序机制

1. **localStorage 存储**：key 为 `bama_website_clicks`，值为 `{ [siteId]: number }` 的 JSON
2. **点击时记录**：用户点击卡片链接时，`siteId` 对应计数 +1
3. **排序逻辑**：
   - 默认排序：按 `tag` 权重（热门 > 官方 > 免费 > 无标签），同权重按原始顺序
   - 个性化排序：有访问记录时，按点击次数降序排；无记录的按默认序
   - 分类内排序，不跨分类
4. **"常用"标签**：访问次数 >= 3 的网站卡片右上角显示"常用"角标

## 假设与决策

1. **不新增路由**：增强在首页内完成，不创建独立页面
2. **localStorage 存储上限**：仅存 siteId -> count，数据极小，无性能隐患
3. **不引入外部数据库**：轻量方案，全部本地存储
4. **分类固定**：8 大分类由数据文件定义，暂不支持用户自定义分类
5. **图标采用 SVG 内联**：与现有模式一致，不引入图标库
6. **移动端**：分类筛选按钮横向滚动，卡片网格自动响应式

## 验证步骤

1. 首页加载后确认所有分类和网站卡片正确渲染
2. 点击分类筛选按钮，确认只显示对应分类的网站
3. 点击"全部"按钮，确认恢复所有网站展示
4. 点击某个网站链接，确认跳转正确且 localStorage 中计数增加
5. 多次点击同一网站（>=3次），确认出现"常用"角标
6. 刷新页面，确认访问频率排序生效
7. 切换 dark mode，确认所有样式正确
8. 移动端视口下确认布局正常
9. `npm run build` 构建验证通过
