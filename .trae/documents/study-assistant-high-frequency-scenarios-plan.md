# AI学习助手高频使用场景规划

## 摘要

当前 AI学习助手已经具备较好的工具集合基础：数学口算/计算题、语文拼音/字帖/古诗、英语单词拼读，并且已经有“最近 / 常用工具”入口和点击排序能力。

下一步建议不要盲目堆工具，而是围绕家长真正高频的辅导任务做“场景化入口”和“练习闭环”。最值得优先规划的方向是：

1. 每日口算 10 分钟
2. 错题重练生成器
3. 一键打印今日练习包
4. 生字听写 / 默写纸
5. 古诗背诵与默写助手
6. 英语单词听写

这些场景都能基于当前已有工具扩展，落地成本相对可控，且非常贴近家长日常使用。

## 当前状态分析

### 现有入口页

文件：`src/views/StudyAssistantView.vue`

当前能力：
- 页面标题：`AI学习助手`
- 顶部展示「最近 / 常用工具」
- 按学科分区展示工具：数学 / 语文 / 英语
- 侧边导航通过锚点滚动到对应分区
- 点击工具会记录使用次数，用于常用排序

### 现有工具数据

文件：`src/data/studyTools.js`

当前工具：

数学：
- `20以内题目生成器` → `/math-within-20`
- `百以内题目生成器` → `/math-within-100`
- `三位数加减题目生成器` → `/math-three-digit`
- `百数表生成` → `/hundred-chart`
- `百以内+三位数加减混合` → `/mixed-math`

语文：
- `古诗词讲解速查` → `/ancient-poetry`
- `拼音标注工具` → `/pinyin-annotation`
- `生字字帖生成器` → `/character-practice`

英语：
- `单词拼读` → `/word-pronunciation`

### 现有点击统计

文件：`src/composables/useStudyToolClicks.js`

当前能力：
- 使用 `localStorage` 保存学习工具点击次数
- 点击越多越靠前
- 点击达到 3 次后显示常用星标

### 当前主要短板

1. 多数工具停留在“生成 / 查询 / 打印”，还没有形成“练习 → 批改 → 错题 → 再练”的闭环。
2. “AI学习助手”的 AI 感较弱，目前更像工具集合页。
3. 家长高频任务没有被单独建模，例如“今日口算”“错题再练”“打印一套练习”。
4. 英语模块较薄，只有单词拼读。
5. 数学题目大多已有答案字段，但没有统一的答案页、批改、错题能力。
6. 语文工具之间还未打通，例如生字字帖不能直接生成听写纸，古诗速查不能生成默写纸。

## 高频场景规划

## 场景一：每日口算 10 分钟

### 用户场景
家长每天晚上想快速给孩子打印一页口算题，要求步骤少、题量合适、可打印、最好能带答案。

### 适用人群
- 小学一年级到三年级家长
- 高频程度：非常高

### 推荐优先级
高，建议第一批落地。

### 现有基础
可复用：
- `src/views/MathWithin20View.vue`
- `src/views/MathWithin100View.vue`
- `src/views/MathThreeDigitView.vue`
- `src/views/MixedMathView.vue`

尤其是 `MixedMathView.vue` 已经非常接近“每日口算”场景。

### 建议功能
新增一个场景入口：`每日口算 10 分钟`

可以先做轻量版：
- 年级选择：一年级 / 二年级 / 三年级
- 题量选择：20题 / 30题 / 50题
- 难度选择：基础 / 进阶 / 易错混合
- 一键生成
- 一键打印
- 打印页包含：姓名、日期、用时、得分、家长签字

后续增强：
- 答案页
- 屏幕答题
- 自动批改
- 错题再练

### 建议落地方式
第一阶段不新建复杂题库，优先在学习助手首页加一个场景卡片，跳转到现有 `/mixed-math`。
第二阶段新建 `DailyMathView.vue`，整合现有数学生成逻辑。

## 场景二：错题重练生成器

### 用户场景
孩子作业错了几道计算题，家长希望输入错题后，自动生成同类题再练。

### 适用人群
- 小学低中年级家长
- 高频程度：高

### 推荐优先级
高，建议第二批落地。

### 现有基础
数学工具中已有大量规则生成逻辑：
- 20以内进位 / 退位
- 百以内加减
- 三位数进位 / 退位
- 混合题生成

### 建议功能
新建工具：`错题重练生成器`

输入示例：
```text
36+27=
84-39=
204+398=
```

初版规则识别：
- 判断加法 / 减法
- 判断数值范围：20以内、百以内、三位数
- 判断是否进位 / 退位
- 每道错题生成 5-10 道同类题

输出：
- 同类题列表
- 答案
- 打印练习纸

### 建议落地方式
新增页面：`src/views/WrongProblemPracticeView.vue`
新增路由：`/wrong-problem-practice`
新增学习工具数据：`studyTools.js` 中数学分类新增入口。

### 价值
这是最接近“AI学习助手”的能力之一，即使初版不用大模型，也能体现“根据错题智能出题”。

## 场景三：一键打印今日练习包

### 用户场景
家长周末或每天晚上想一次性打印一套练习，包括口算、生字、拼音、古诗、英语单词。

### 适用人群
- 小学低年级家长
- 高频程度：高

### 推荐优先级
中高，建议在每日口算和语文听写之后做。

### 现有基础
可组合已有工具：
- `/mixed-math` 口算
- `/character-practice` 生字字帖
- `/pinyin-annotation` 拼音标注
- `/ancient-poetry` 古诗
- `/word-pronunciation` 英语单词

### 建议功能
新增场景：`今日练习包`

配置项：
- 是否包含口算
- 是否包含生字字帖
- 是否包含拼音练习
- 是否包含古诗默写
- 是否包含英语单词听写

输出：
- 多页打印内容
- 每页统一页眉：姓名、日期、用时、得分
- 支持一键打印

### 建议落地方式
不要第一阶段就做完整 PDF 合并；先做一个独立页面，按用户选择生成多个打印 section，然后统一 `window.print()`。

新增页面：`src/views/DailyPracticePackView.vue`
新增路由：`/daily-practice-pack`
新增学习工具数据：放在数学或新增“综合”分类需要另行讨论。

## 场景四：生字听写 / 默写纸

### 用户场景
家长每天帮孩子听写生字词，希望快速生成看拼音写汉字、田字格默写、组词练习。

### 适用人群
- 小学低年级家长
- 高频程度：非常高

### 推荐优先级
高，建议第一批或第二批落地。

### 现有基础
可复用：
- `src/views/CharacterPracticeView.vue`
- `/api/data/characters`
- `pinyin-pro`

`CharacterPracticeView.vue` 已经支持年级 / 课文 / 生字选择，这是非常好的基础。

### 建议功能
可以在现有生字字帖工具中新增模式，也可以新建工具。

建议新增独立工具：`生字听写纸`

功能：
- 按年级 / 课文选择生字
- 自由输入生字或词语
- 生成以下练习：
  - 看拼音写汉字
  - 看汉字写拼音
  - 田字格默写
  - 听写记录表
- 打印页包含姓名、日期、得分、错字订正栏

### 建议落地方式
新增页面：`src/views/CharacterDictationView.vue`
复用 `CharacterPracticeView.vue` 的数据加载逻辑和拼音能力。

新增路由：`/character-dictation`
新增学习工具数据：语文分类新增 `生字听写纸`。

## 场景五：古诗背诵与默写助手

### 用户场景
孩子要背古诗，家长希望能快速查看解释，并打印默写纸或挖空练习。

### 适用人群
- 小学到初中家长
- 高频程度：高

### 推荐优先级
中高。

### 现有基础
可复用：
- `src/views/AncientPoetryView.vue`
- `src/data/poems.js`

现有古诗工具已经有原文、译文、赏析、重点字词，这是非常好的内容基础。

### 建议功能
可以在现有古诗工具中增加“练习模式”：
- 原文默写纸
- 挖空默写
- 诗句排序
- 作者朝代填空
- 重点字词解释填空
- 隐藏译文 / 隐藏原文用于背诵

### 建议落地方式
优先在 `AncientPoetryView.vue` 内新增打印练习模式，不必新建页面。

好处：
- 用户查到诗后直接生成练习
- 不需要重复选择古诗
- 对现有工具增强明显

## 场景六：英语单词听写

### 用户场景
家长输入一组英语单词，工具自动朗读，孩子听写，系统判断对错或打印听写纸。

### 适用人群
- 小学英语家长
- 高频程度：中高

### 推荐优先级
中高。

### 现有基础
可复用：
- `src/views/WordPronunciationView.vue`
- 浏览器 Web Speech API

当前单词拼读已支持输入单词、去重、顺序播放、循环播放。

### 建议功能
新增工具：`英语单词听写`

功能：
- 输入单词表
- 自动朗读单词
- 孩子输入拼写
- 自动判断正确 / 错误
- 错词标记
- 打印听写纸
- 支持随机顺序

### 建议落地方式
第一阶段可以直接增强 `WordPronunciationView.vue`：增加“听写模式”。
如果功能复杂，再拆成 `WordDictationView.vue`。

## 场景七：百数表找规律练习

### 用户场景
一年级家长希望通过百数表训练数感，比如找奇偶数、5的倍数、缺失数、跳数。

### 适用人群
- 一年级家长
- 高频程度：中

### 现有基础
可复用：
- `src/views/HundredChartView.vue`

现有百数表已有高亮和自定义范围能力。

### 建议功能
增强现有百数表：
- 缺失数练习
- 跳数练习：2、5、10
- 圈出指定规律数字
- 打印题目说明
- 打印答案页

### 建议落地方式
在 `HundredChartView.vue` 增加“练习模式”。

## 场景八：家长批改助手

### 用户场景
孩子在线做完题，家长或系统快速批改，得到正确率和错题列表。

### 适用人群
- 小学数学家长
- 高频程度：中

### 现有基础
数学题目对象多数已有 `answer` 字段。

### 建议功能
先从数学工具增强：
- 屏幕答题
- 检查答案
- 正确率
- 错题列表
- 一键错题再练

### 建议落地方式
优先改 `MathWithin20View.vue`，因为题型简单，风险最低。
验证模式成熟后，再推广到 `MixedMathView.vue` 和 `MathThreeDigitView.vue`。

## 建议优先级排序

### P0：最值得优先做
1. 每日口算 10 分钟
2. 生字听写 / 默写纸
3. 错题重练生成器

原因：
- 家长高频
- 和现有能力贴合
- 能明显提升“助手感”
- 对日常留存有帮助

### P1：第二阶段做
4. 古诗背诵与默写助手
5. 英语单词听写
6. 一键打印今日练习包

原因：
- 价值高，但依赖更多交互和打印模板整合

### P2：后续增强
7. 百数表找规律练习
8. 家长批改助手
9. 每周学习小结

原因：
- 有价值，但不是当前最短路径
- 需要更统一的数据记录和题目模型

## 建议第一轮落地方案

如果只做一轮，建议选择 3 件事：

### 1. 在学习助手页增加“高频场景”分区

位置：`src/views/StudyAssistantView.vue`

注意：这次不是恢复之前那种简单“家长高频”工具卡，而是更明确的“场景任务”入口。

建议标题：`高频学习场景`

入口：
- 每日口算 10 分钟
- 生字听写纸
- 错题重练
- 古诗默写
- 英语听写

其中未完成的功能可以先标记为“规划中”，但建议只展示真实可点击的入口，避免造成落差。

### 2. 新增 `生字听写纸`

原因：
- 与当前 `CharacterPracticeView.vue` 复用度最高
- 家长使用频率非常高
- 适合低年级用户
- 不依赖复杂 AI 能力

建议新增：
- `src/views/CharacterDictationView.vue`
- `/character-dictation`
- `studyTools.js` 语文分类入口

### 3. 新增 `错题重练生成器`

原因：
- 最能体现 AI学习助手价值
- 初版可以用规则实现，不依赖大模型
- 与数学工具互补

建议新增：
- `src/views/WrongProblemPracticeView.vue`
- `/wrong-problem-practice`
- `studyTools.js` 数学分类入口

## 不建议当前立即做的内容

1. 不建议一开始就接入大模型讲题，成本和稳定性较高。
2. 不建议马上做账号体系和云端学习记录。
3. 不建议一次性把所有数学工具重构成统一题库模型。
4. 不建议展示不可点击的“规划中”入口太多，会影响体验。
5. 不建议改动现有左侧导航锚点体验，用户已明确喜欢该模式。

## 验证思路

后续如果执行第一轮落地，需要验证：

1. `npm run build` 构建通过。
2. `/study-assistant` 页面仍保留数学 / 语文 / 英语锚点滚动。
3. 新增场景入口在移动端不溢出。
4. 新增工具能从学习助手页进入。
5. 新增工具点击后能进入最近 / 常用工具排序。
6. 打印页面在浏览器打印预览中正常。
7. 不影响已有数学、字帖、拼音、古诗、英语工具。

## 总结

AI学习助手下一阶段最值得做的不是继续堆普通工具，而是把已有工具包装成家长日常真正会反复使用的学习场景。

建议优先落地：
- `每日口算 10 分钟`
- `生字听写 / 默写纸`
- `错题重练生成器`

这三个方向最贴近日常陪娃学习，也最能让 AI学习助手从“工具集合”变成“家长每天会打开的学习工作台”。
