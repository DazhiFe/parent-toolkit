# AI学习助手页面高频家长场景优化计划

## 摘要
当前 `AI学习助手` 页面已经聚合了数学、语文、英语三类学习工具，但整体仍偏“工具列表”，缺少家长高频使用时最需要的：快速入口、最近/常用排序、年级/场景提示、打印能力标识、占位工具处理。

建议本轮优先做一版轻量但实用的优化：保留现有“左侧导航点击，右侧锚点滚动”的体验，同时增加“家长高频入口”和“最近常用工具”，并把页面改为数据驱动，便于后续继续扩展。

## 当前状态分析

### 已有页面结构
文件：`src/views/StudyAssistantView.vue`

当前页面包含：
- 顶部标题：`学习助手`
- 左侧导航：来自 `SideNav.vue`，包含 `数学 / 语文 / 英语`
- 右侧内容：三个 section，分别是：
  - `#math`
  - `#chinese`
  - `#english`
- 每个 section 中手写工具卡片列表

### 当前工具列表
数学：
- 20以内题目生成器
- 百以内题目生成器
- 三位数加减题目生成器
- 百数表生成
- 百以内+三位数加减混合

语文：
- 古诗词讲解速查
- 拼音标注工具
- 生字字帖生成器

英语：
- 单词记忆（当前是 `#` 占位链接，不可用）
- 单词拼读

### 当前不足
1. 工具数据硬编码在 `StudyAssistantView.vue`，扩展不方便。
2. 没有“最近使用 / 常用工具”，家长每次都要重新找。
3. 没有“家长高频入口”，比如“生成口算”“打印字帖”“查古诗”。
4. 没有年级/场景/能力标签，家长不容易快速判断适不适合自己孩子。
5. “单词记忆”是占位链接，点击后没有实际功能，体验不好。
6. 页面标题是“学习助手”，顶部导航是“AI学习助手”，命名略不统一。
7. 数学/语文/英语三个 section 模板重复度高。

## 建议本轮实施范围

### 目标
在不大改路由、不新增复杂 AI 接口的前提下，把页面从“静态工具列表”升级为“家长高频学习工具工作台”。

### 保留体验
- 保留左侧导航：数学 / 语文 / 英语
- 保留右侧 section 锚点滚动
- 保留现有工具卡片视觉风格
- 不改变现有具体工具页路由

## 拟改动文件

### 1. 新增 `src/data/studyTools.js`

#### 做什么
把学习助手的分类和工具数据从 `StudyAssistantView.vue` 抽离到数据文件。

#### 为什么
后续新增工具、改描述、加标签不需要改页面模板。

#### 计划结构
```js
export const studyToolCategories = [
  { id: 'math', name: '数学', icon: 'calculator' },
  { id: 'chinese', name: '语文', icon: 'book' },
  { id: 'english', name: '英语', icon: 'language' },
]

export const studyTools = [
  {
    id: 'math-within-20',
    name: '20以内题目生成器',
    description: '生成20以内的加减法练习题',
    url: '/math-within-20',
    category: 'math',
    color: 'cyan',
    icon: 'calculator',
    tags: ['可打印', '口算'],
    grades: ['一年级'],
    frequent: true,
  }
]
```

#### 工具数据调整
- 保留全部真实可用工具。
- `单词记忆` 当前没有真实页面，先从可点击工具列表中移除，避免误导用户。
- 继续保留 `单词拼读` 作为英语模块核心入口。

### 2. 新增 `src/composables/useStudyToolClicks.js`

#### 做什么
为学习助手工具增加本地点击统计能力。

#### 为什么
家长高频使用往往集中在几个工具：口算题、字帖、拼音、古诗、单词拼读。点击统计后可以自动显示“最近/常用”。

#### 实现方式
参考 `src/composables/useAiToolClicks.js`，但使用独立 localStorage key：
- `bama_studytool_clicks`
- `bama_studytool_hidden`（本轮可预留，不强制做管理面板）

本轮需要的方法：
- `recordClick(toolId)`
- `getClickCount(toolId)`
- `isFrequent(toolId)`
- `getSortedTools(tools)`

### 3. 修改 `src/views/StudyAssistantView.vue`

#### 3.1 数据驱动渲染
从：
- `mathTools`
- `chineseTools`
- `englishTools`

改为：
- `studyToolCategories`
- `studyTools`
- `toolsByCategory`

每个分类自动渲染 section，section id 保持 `math/chinese/english`，保证 `SideNav.vue` 不需要大改。

#### 3.2 新增“家长高频入口”区域
在页面标题下方新增一块紧凑区域：

标题建议：`家长高频`

默认展示 5 个快捷入口：
1. 今日口算 → `/mixed-math`
2. 打印生字字帖 → `/character-practice`
3. 古诗词速查 → `/ancient-poetry`
4. 给课文标拼音 → `/pinyin-annotation`
5. 英语单词拼读 → `/word-pronunciation`

这些入口都指向已有工具，不新增路由。

#### 3.3 新增“最近/常用工具”区域
如果用户已有点击记录：
- 展示点击次数最高的前 4 个学习工具

如果没有点击记录：
- 展示默认推荐的前 4 个高频工具：
  - 百以内+三位数加减混合
  - 生字字帖生成器
  - 拼音标注工具
  - 古诗词讲解速查

#### 3.4 工具卡片增加标签
每张卡片展示 1-3 个标签，例如：
- `可打印`
- `口算`
- `字帖`
- `拼音`
- `查询`
- `听读`

标签使用小号圆角样式，放在描述下方或右上角，避免卡片过重。

#### 3.5 点击工具时记录使用次数
所有 `RouterLink` 增加点击处理：
- `@click="handleToolClick(tool.id)"`

用于驱动“最近/常用工具”。

#### 3.6 文案统一
页面标题改为：`AI学习助手`

描述改为：
`为孩子提供口算练习、语文打印、古诗速查和英语听读等高频学习工具`

### 4. 修改 `src/components/AppHeader.vue`

#### 做什么
移动端菜单文案从 `学习助手` 统一为 `AI学习助手`。

#### 为什么
当前桌面端是 `AI学习助手`，移动端是 `学习助手`，统一可降低认知差异。

### 5. 可选但建议修改 `src/components/SideNav.vue`

#### 做什么
左侧导航仍保留：
- 数学
- 语文
- 英语

不新增“高频入口”锚点，避免打破用户熟悉的左侧导航结构。

#### 为什么
用户之前明确喜欢“左侧导航点击，右侧内容锚点滚动”的体验。本轮不改变核心导航模型。

## 不做内容
本轮不做以下内容，避免过度开发：

1. 不新增真正的 AI 对话接口。
2. 不新增“单词记忆”完整功能页。
3. 不新增错题本、学习计划、每日任务等复杂状态系统。
4. 不改变现有工具页路由结构。
5. 不把学习助手嵌入首页大改版。

## 预期效果

优化后，家长进入页面后会先看到：
1. 最常用的快捷入口。
2. 最近/常用工具。
3. 数学、语文、英语完整分区。

高频操作路径会明显缩短：
- 原来：进入页面 → 找数学 → 找混合计算 → 点击
- 优化后：进入页面 → 点击“今日口算”

- 原来：进入页面 → 找语文 → 找字帖 → 点击
- 优化后：进入页面 → 点击“打印生字字帖”

## 验证步骤

1. 运行 `npm run build`，确保构建通过。
2. 打开 `/study-assistant`，检查页面标题、快捷入口、常用工具区域、三大学科分区是否显示正常。
3. 点击快捷入口，确认能跳转到对应工具页。
4. 返回 `/study-assistant` 后，确认点击过的工具会进入最近/常用工具排序。
5. 点击左侧导航“数学 / 语文 / 英语”，确认仍可平滑滚动到对应 section。
6. 在移动端宽度下检查卡片和快捷入口不会横向溢出。
